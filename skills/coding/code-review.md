# 代码审查技能 (Code Review)

## 描述
分析代码质量、发现潜在问题、提供优化建议

## 使用场景
- 完成代码编写后需要质量检查
- 接收他人代码需要评估
- 重构前需要了解代码问题
- 学习优秀代码实践

## 审查维度

### 1. 代码质量
- [ ] 命名是否清晰有意义
- [ ] 函数是否单一职责
- [ ] 代码重复度 (DRY 原则)
- [ ] 注释是否恰当 (解释为什么，而非做什么)

### 2. 潜在问题
- [ ] 边界条件处理
- [ ] 错误处理和异常
- [ ] 空值和 null 检查
- [ ] 资源泄漏风险
- [ ] 并发安全问题

### 3. 性能优化
- [ ] 时间复杂度分析
- [ ] 空间复杂度分析
- [ ] 不必要的计算
- [ ] 缓存机会
- [ ] I/O 优化

### 4. 可维护性
- [ ] 代码结构清晰
- [ ] 模块划分合理
- [ ] 依赖关系清晰
- [ ] 测试覆盖度

### 5. 安全性
- [ ] 输入验证
- [ ] SQL 注入风险
- [ ] XSS 风险
- [ ] 敏感信息处理
- [ ] 权限检查

## 审查流程

1. **理解上下文** - 代码用途、业务场景
2. **静态分析** - 阅读代码结构和逻辑
3. **问题识别** - 标记潜在问题
4. **优先级排序** - 严重/重要/建议
5. **提供建议** - 具体可执行的改进方案

## 输出格式

```markdown
## 📊 代码审查报告

### ✅ 优点
- [列出代码做得好的地方]

### 🔴 严重问题 (必须修复)
1. **问题描述**
   - 位置：`文件：行号`
   - 风险：[说明潜在影响]
   - 建议：[具体修复方案]

### 🟡 重要问题 (建议修复)
1. **问题描述**
   - 位置：`文件：行号`
   - 建议：[改进方案]

### 🟢 优化建议 (可选)
1. **问题描述**
   - 建议：[最佳实践]

### 📈 总体评分
- 代码质量：X/10
- 可维护性：X/10
- 性能：X/10
- 安全性：X/10
```

## 示例对话

**用户**: 帮我审查一下这个函数

```python
def get_user_data(user_id):
    query = "SELECT * FROM users WHERE id = " + user_id
    return db.execute(query)
```

**助手**:

## 📊 代码审查报告

### 🔴 严重问题 (必须修复)

1. **SQL 注入漏洞**
   - 位置：`get_user_data:2`
   - 风险：攻击者可通过构造恶意 user_id 获取任意数据
   - 建议：使用参数化查询

```python
def get_user_data(user_id):
    query = "SELECT * FROM users WHERE id = ?"
    return db.execute(query, (user_id,))
```

### 🟡 重要问题 (建议修复)

2. **缺少输入验证**
   - 位置：`get_user_data:1`
   - 建议：验证 user_id 类型和范围

3. **缺少错误处理**
   - 建议：添加 try-except 处理数据库异常

### 🟢 优化建议

4. **使用类型注解**
   - 建议：添加 Python type hints 提高可读性

### 📈 总体评分
- 代码质量：4/10
- 可维护性：5/10
- 性能：7/10
- 安全性：1/10 ⚠️
