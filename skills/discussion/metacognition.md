# 元认知引导技能 (Metacognition Guidance)

## 描述
引导对思考本身的思考，提升思维质量和学习效率

## 使用场景
- 学习后反思
- 决策后复盘
- 思维过程检视
- 认知偏见识别

## 元认知层次

### Level 1: 认知监控
```
我在想什么？
我的注意力在哪里？
我理解了吗？
我记住什么了？
```

### Level 2: 认知评估
```
我的理解准确吗？
我的记忆可靠吗？
我的推理严谨吗？
我的判断公正吗？
```

### Level 3: 认知调节
```
我需要调整策略吗？
我应该换个角度吗？
我需要更多信息吗？
我是否被偏见影响？
```

### Level 4: 认知哲学
```
知识的本质是什么？
我如何知道我知道？
思维的局限在哪里？
什么是好的思考？
```

## 元认知问题库

### 学习过程中的元认知

#### 学习前
```
- 我为什么要学这个？
- 我已经知道什么？
- 我的学习目标是什么？
- 什么策略最适合我？
- 预计会遇到什么困难？
```

#### 学习中
```
- 我理解了吗？
- 我能用自己的话复述吗？
- 这和已知的有什么联系？
- 我是否走神了？
- 需要放慢/加快吗？
```

#### 学习后
```
- 我学到了什么？
- 哪些还不清楚？
- 如何检验掌握程度？
- 何时复习？
- 如何应用？
```

### 决策过程中的元认知

#### 决策前
```
- 这真的是问题所在吗？
- 我的目标是什么？
- 我有哪些选项？
- 我基于什么信息？
- 我可能有哪些偏见？
```

#### 决策中
```
- 我的推理过程是什么？
- 我是否情绪化？
- 我是否过度自信？
- 我考虑了反面证据吗？
- 如果是别人，会怎么决定？
```

#### 决策后
```
- 结果如何？
- 和预期一致吗？
- 什么做对了？
- 什么可以改进？
- 下次会怎么做？
```

### 问题解决中的元认知

#### 问题定义
```
- 这真的是问题吗？
- 问题的核心是什么？
- 问题是否被正确表述？
- 重新定义问题会怎样？
```

#### 解决过程
```
- 我用的策略有效吗？
- 是否陷入思维定势？
- 是否有更好的方法？
- 我是否过早下结论？
```

#### 解决后
```
- 方案真的解决问题了吗？
- 有副作用吗？
- 可以推广吗？
- 学到了什么通用方法？
```

## 元认知工具

### 1. 思考日志
```markdown
# 思考日志 - [日期]

## 情境
[在什么情况下思考]

## 思考内容
[想了什么]

## 思考过程
[如何思考的]

## 情绪状态
[当时的情绪]

## 偏见检查
[可能的认知偏见]

## 质量评估
[思考质量如何]

## 改进方向
[下次如何改进]
```

### 2. 思维检查清单
```
决策前检查:
- [ ] 明确目标
- [ ] 收集充分信息
- [ ] 考虑多个选项
- [ ] 检查偏见
- [ ] 咨询他人意见
- [ ] 考虑长期影响
- [ ] 设定决策标准
```

### 3. 认知偏见自查表
```
常见偏见检查:

确认偏误:
- 我是否只找支持自己的证据？
- 我是否忽略反面证据？

锚定效应:
- 我是否过度依赖第一个信息？
- 如果换个起点，结论会不同吗？

可得性启发:
- 我是否高估了容易想到的例子？
- 统计数据是什么？

过度自信:
- 我有多确定？
- 我可能错在哪里？
- 什么证据会改变我的想法？
```

### 4. 反思提示卡
```
每日反思:
- 今天最好的决策是什么？
- 今天最差的决策是什么？
- 学到了什么？
- 明天如何改进？

每周反思:
- 这周的模式是什么？
- 什么思维习惯有用？
- 什么需要改变？
- 下周重点是什么？
```

## 元认知训练方法

### 方法 1: 思考出声 (Think Aloud)
```
边思考边说出想法:
- "我在想..."
- "我假设..."
- "我困惑的是..."
- "我意识到..."

好处：让思维过程显性化
```

### 方法 2: 双栏笔记
```
左栏：内容/观点
右栏：我的思考过程

示例:
左：机器学习需要数据
右：我为什么相信这个？因为...
```

### 方法 3: 第三人称视角
```
把自己当观察者:
- "小明在这种情况下会怎么想？"
- "如果是旁观者，会怎么看？"
- "我的思考有什么模式？"

好处：减少情绪干扰
```

### 方法 4: 预演与复盘
```
事前预演:
- 想象思考过程
- 预测可能的问题
- 准备应对策略

事后复盘:
- 回顾实际过程
- 对比预期与现实
- 提取经验教训
```

## 元认知对话模式

### 模式 1: 引导反思
```
助手：刚才的讨论中，你觉得自己思考得怎么样？

用户：还行吧

助手：具体来说：
- 哪个时刻你觉得最清晰？
- 哪个时刻觉得最困惑？
- 如果重来，会有什么不同？
```

### 模式 2: 偏见检查
```
助手：我注意到你很确定这个结论。
能说说你的推理过程吗？

用户：[解释]

助手：好的。现在让我问几个挑战性问题：
- 什么证据会让你改变想法？
- 有没有考虑相反的情况？
- 如果是你反对的观点，会怎么论证？
```

### 模式 3: 策略调整
```
助手：这个学习方法效果如何？

用户：感觉一般

助手：具体来说：
- 什么部分有效？
- 什么部分无效？
- 如果调整策略，会试什么？
- 如何检验新策略的效果？
```

## 示例对话

**用户**: 我觉得我学习效率很低，怎么办？

**助手**:

让我帮你用元认知的方式分析这个问题。

## 🔍 元认知分析：学习效率

### 第一步：澄清问题

**问题**: "效率低"具体指什么？
- 学得慢？
- 记不住？
- 无法应用？
- 容易分心？

能具体描述吗？

### 第二步：监控当前状态

**请你观察自己**:
- 学习时注意力在哪里？
- 什么情况下最容易分心？
- 学习时的感受是什么？(焦虑/无聊/困惑)
- 你如何判断自己"学会了"？

### 第三步：评估策略

**当前方法**:
- 你怎么学习的？(看书/视频/实践)
- 学习多久休息一次？
- 如何复习？
- 如何检验掌握程度？

**效果评估**:
- 哪些方法有效？
- 哪些无效？
- 为什么？

### 第四步：识别模式

**可能的模式**:
- 是否追求完美导致进度慢？
- 是否缺乏明确目标？
- 是否没有及时反馈？
- 是否方法不适合学习风格？

### 第五步：策略调整

**建议尝试**:

1. **目标具体化**
   - 不是"学机器学习"
   - 而是"今天理解线性回归"

2. **主动 recall**
   - 不是反复阅读
   - 而是盖上书回忆

3. **间隔复习**
   - 不是集中学习
   - 而是分散多次

4. **费曼技巧**
   - 尝试教给别人
   - 发现理解漏洞

5. **元认知监控**
   - 每 30 分钟问自己：
   - "我理解了吗？"
   - "需要调整吗？"

### 第六步：实验与反馈

**建议**:
选一个策略，试一周，然后：
- 记录效果
- 调整优化
- 再试另一个

**关键**: 把自己当实验对象，
持续观察、调整、优化。
