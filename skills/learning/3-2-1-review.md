# 3-2-1 回顾法技能

## 描述
结构化回顾学习内容，强化记忆和理解

## 使用场景
- 学习后立即回顾
- 每日/每周复习
- 准备考试复习
- 整理学习笔记

## 3-2-1 结构

### 3 个要点 (3 Key Points)
**目的**: 提取核心信息

**问题**:
- 最重要的 3 个概念是什么？
- 必须记住的 3 个事实是什么？
- 最关键的 3 个步骤是什么？

**要求**:
- 限制在 3 个 (强迫优先级排序)
- 用自己的话表述
- 简洁清晰 (每点 1-2 句话)

**示例**:
```
主题：梯度下降

3 个要点:
1. 梯度下降是找函数最小值的迭代方法
2. 每次沿负梯度方向更新参数
3. 学习率控制步长，太大太小都不好
```

### 2 个例子 (2 Examples)
**目的**: 连接抽象和具体

**问题**:
- 能举 2 个实际应用的例子吗？
- 2 个生活中的类比是什么？
- 2 个代码实现的例子？

**要求**:
- 不同类型/场景的例子
- 一个简单，一个复杂
- 最好有一个自己创造的

**示例**:
```
主题：梯度下降

2 个例子:
1. 下山类比：在雾中下山，每次往最陡方向走一步
2. 机器学习：训练线性回归，找最佳拟合直线
```

### 1 个问题 (1 Question)
**目的**: 发现知识漏洞，引导深入思考

**问题**:
- 还有什么不明白的？
- 什么情况下这个方法不适用？
- 这和之前学的 X 有什么关系？

**要求**:
- 诚实面对不懂的地方
- 可以是深入思考的问题
- 指导下一步学习

**示例**:
```
主题：梯度下降

1 个问题:
- 梯度下降一定能找到全局最小值吗？
  如果不是，怎么避免局部最优？
```

## 完整模板

```markdown
# 3-2-1 回顾：[主题]

**日期**: YYYY-MM-DD
**学习时间**: X 分钟/小时

## 3️⃣ 三个要点

1. **[要点 1]**
   [简洁表述]

2. **[要点 2]**
   [简洁表述]

3. **[要点 3]**
   [简洁表述]

## 2️⃣ 两个例子

**例子 1 (简单)**:
[简单例子或类比]

**例子 2 (应用)**:
[实际应用或代码]

## 1️⃣ 一个问题

**问题**: [你的疑问]

**思考**: [你的初步想法]

**下一步**: [如何找到答案]
```

## 变化形式

### 5-3-1 (深度学习)
```
5 个要点 - 更全面
3 个例子 - 更多联系
1 个问题 - 深度思考

适用：复杂主题，需要更深入理解
```

### 3-2-1-Action (实践导向)
```
3 个要点
2 个例子
1 个问题
+ 1 个行动 (下一步做什么)

适用：技能学习，需要立即实践
```

### 3-2-1-Share (教学导向)
```
3 个要点
2 个例子
1 个问题
+ 如何教给别人

适用：准备分享/教学
```

## 应用场景

### 场景 1: 课后立即回顾
```
时机：学习结束后 5 分钟内
时间：5-10 分钟
目的：巩固短期记忆

流程:
1. 合上书本/笔记
2. 回忆并写下 3-2-1
3. 对照笔记检查
4. 标记遗漏和错误
```

### 场景 2: 每日复习
```
时机：每天学习结束时
时间：15-20 分钟
目的：整合当天内容

流程:
1. 回顾每个主题的 3-2-1
2. 找出主题间的联系
3. 更新问题列表
4. 规划明天学习
```

### 场景 3: 每周总结
```
时机：周末
时间：30-60 分钟
目的：系统整理

流程:
1. 收集本周所有 3-2-1
2. 提炼本周核心 (3 个)
3. 整合例子库
4. 回答部分问题
5. 生成新问题
```

### 场景 4: 考前复习
```
时机：考试前
时间：根据科目数量
目的：快速 recall

流程:
1. 只看 3-2-1 笔记
2. 尝试回忆细节
3. 重点复习问题
4. 自测掌握程度
```

## 进阶技巧

### 1. 间隔重复
```
第 1 次：学习后立即
第 2 次：1 天后
第 3 次：3 天后
第 4 次：7 天后
第 5 次：14 天后
第 6 次：30 天后

每次回顾时:
- 先看之前的 3-2-1
- 尝试回忆
- 更新/补充
```

### 2. 主动 recall
```
❌ 被动：重读笔记
✅ 主动：盖上笔记，尝试写出 3-2-1

效果：主动 recall 比重读效果好 5 倍
```

### 3. 交叉对比
```
对比不同时间的 3-2-1:
- 理解是否加深？
- 表述是否更清晰？
- 问题是否已解决？

示例:
第 1 天的要点 vs 第 7 天的要点
→ 看理解的演进
```

### 4. 同伴分享
```
1. 写好自己的 3-2-1
2. 和同伴交换
3. 互相讲解
4. 补充对方的要点

效果：教学相长
```

## 常见问题

### Q1: 想不出 3 个要点怎么办？
```
A: 
- 说明没理解核心，回去重学
- 或者主题太大，拆分成小主题
- 先写 2 个，再逼自己想第 3 个
```

### Q2: 例子太难想怎么办？
```
A:
- 从简单类比开始
- 搜索实际应用
- 自己创造简化场景
- 参考教材/网络的例子
```

### Q3: 没有问题怎么办？
```
A:
- 可能理解不够深，追问"为什么"
- 问"如果...会怎样"
- 问"这和 X 有什么关系"
- 问"这个方法的局限是什么"
```

### Q4: 3-2-1 写完后做什么？
```
A:
- 保存好，定期回顾
- 回答问题 (查资料/问人)
- 应用到实践
- 分享给他人
```

## 示例对话

**助手**: 我们来用 3-2-1 回顾今天学的支持向量机 (SVM)。

请你先自己回想一下，然后告诉我：
- 3 个最重要的要点
- 2 个例子 (类比或应用)
- 1 个你还有的疑问

不着急，慢慢想。

**用户**: 
3 个要点:
1. SVM 是找最大间隔的分类器
2. 支持向量是关键的样本点
3. 核技巧可以处理非线性

2 个例子:
1. 用超平面分隔两类点
2. 用 RBF 核处理圆形分布的数据

1 个问题:
核函数具体怎么工作的？为什么能把低维映射到高维？

**助手**:

## ✅ 很好的 3-2-1 回顾！

让我帮你完善一下：

### 📝 完整的 3-2-1

#### 3 个要点

1. **最大间隔分类**
   SVM 不是随便画条线分开两类，而是找间隔最大的那条线
   → 这样泛化能力最强

2. **支持向量决定边界**
   只有边界附近的几个点 (支持向量) 起作用
   其他点可以忽略 → 高效

3. **核技巧处理非线性**
   低维分不开？映射到高维可能就分开了
   不用真的计算高维，用核函数就行

#### 2 个例子

**例子 1 (视觉)**:
想象地上有红蓝两色积木
- 普通分类：随便找根棍子分开
- SVM: 找最宽的通道，两边留足空间

**例子 2 (核技巧)**:
一维分不开：`x` 和 `o` 混在一起
映射到二维：`x²` vs `x`，可能就能画条线分开了

#### 1 个问题

**你的问题**: 核函数怎么工作的？

**简答**: 
核函数 K(x,y) = 计算高维空间的内积，但不用真的映射

**类比**:
想知道两个人在 3D 空间的距离，
不需要真的把他们放到 3D 空间，
用公式直接算就行。

你可以先想想：为什么内积这么重要？
