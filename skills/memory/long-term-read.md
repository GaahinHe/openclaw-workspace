# 长期记忆读取技能 (Long-Term Memory Read)

## 描述
读取 MEMORY.md 中的核心记忆，获取跨会话的持久化信息

## 使用场景
- 会话开始时加载核心上下文
- 需要回忆用户偏好和历史决策
- 查找过往学习进度和记录
- 确认系统配置和技术栈

## 记忆结构

```
MEMORY.md 结构:
├── 核心原则 (Core Principles)
│   ├── 角色定义
│   ├── 沟通风格

├── 系统架构 (System Architecture)
│   ├── 记忆系统

│   └── 决策记录表
│
```

## 读取策略

### 1. 会话启动读取
**时机**: 每次会话开始时自动读取

**内容**:
- 核心原则 (快速回顾角色定位)
- 用户偏好 (确认可用时间段)
- 最近待办 (检查未完成事项)

**目的**: 快速进入状态，保持连续性

### 2. 按需读取
**触发**: 用户提到特定主题

**示例**:
- "我的学习进度如何？" → 读取学习框架 + 待办事项
- "我们之前讨论过 X" → 搜索相关章节
- "系统配置是什么？" → 读取技术栈记录

### 3. 深度读取
**触发**: 需要完整上下文时

**内容**: 全文读取

**场景**:
- 长时间未会话后重新连接
- 用户要求完整回顾
- 重大决策前需要历史背景

## 读取方法

### 方法 1: 全文读取
```python
def read_full_memory():
    """读取完整 MEMORY.md"""
    with open('MEMORY.md', 'r', encoding='utf-8') as f:
        return f.read()
```

**适用**: 会话开始、深度回顾

### 方法 2: 章节读取
```python
def read_memory_section(section_title):
    """读取指定章节"""
    content = read_full_memory()
    # 解析 markdown，提取指定章节
    return extract_section(content, section_title)
```

**适用**: 特定主题查询

### 方法 3: 关键词搜索
```python
def search_memory(keywords):
    """搜索包含关键词的片段"""
    content = read_full_memory()
    matches = []
    for line in content.split('\n'):
        if any(kw in line for kw in keywords):
            matches.append(line)
    return matches
```

**适用**: 模糊记忆检索

## 记忆引用规范

引用记忆时必须标注来源：

```
根据 MEMORY.md 中的记录 [核心原则/角色定义]:
"真诚地帮助，而非表演式帮助"

根据 MEMORY.md 中的记录 [重要决策/2026-02-02]:
"采用三重角色定位：专家/伙伴/老师"
```

## 记忆更新触发

读取时发现以下情况应建议更新：

1. **信息过期**: 
   - 待办事项已完成
   - 技术栈已变更
   - 配置已过时

2. **信息缺失**:
   - 新学习领域未记录
   - 新决策未记录
   - 新偏好未记录

3. **信息冲突**:
   - 多处记录不一致
   - 与当前行为不符

### 2. 3-2-1 回顾法
**结构**:
- **3 个要点**: 最重要的 3 点
- **2 个例子**: 实际应用例子
- **1 个问题**: 你的疑问

### 3. 苏格拉底式提问
**问题类型**:
- 澄清问题：「X 的核心是什么？」
- 检验假设：「这个结论基于什么假设？」
- 探索反面：「如果前提错了会怎样？」
