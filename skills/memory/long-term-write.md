# 长期记忆写入技能 (Long-Term Memory Write)

## 描述
更新 MEMORY.md 中的重要信息，维护精选长期记忆

## 使用场景
- 完成重要学习后记录进度
- 做出重大决策后记录原因
- 发现用户新偏好后记录
- 定期整理和更新记忆

## 写入原则

### 1. 精选原则
**写入前问自己**:
- 这条信息重要吗？
- 跨会话后还需要吗？
- 会影响未来行为吗？
- 值得占用长期记忆空间吗？

**不写入**:
- 临时性信息 (明天就忘的)
- 上下文特定信息 (只在某对话中有用)
- 可推导信息 (能从其他信息推出)

**写入**:
- 用户明确偏好
- 重要决策和原因
- 学习里程碑
- 系统配置变更
- 关系性信息 (信任、边界)

### 2. 结构化原则
**格式要求**:
- 使用 markdown 标题组织
- 使用表格呈现对比
- 使用列表呈现要点
- 添加日期和来源

**示例**:
```markdown
### 决策记录

| 决策 | 日期 | 原因 | 影响 |
|------|------|------|------|
| 采用 X 方案 | 2026-02-24 | 性能更好 | 需要重构 Y |
```

### 3. 可追溯原则
**每条记录包含**:
- 时间戳 (YYYY-MM-DD)
- 来源 (对话/决策/观察)
- 上下文 (为什么重要)

## 写入流程

```
1. 识别记忆候选
   └─→ 用户明确说"记住这个"
   └─→ 重要决策完成
   └─→ 学习里程碑达成
   └─→ 发现新偏好

2. 评估价值
   └─→ 是否符合精选原则？
   └─→ 是否已有类似记录？
   └─→ 是否需要更新旧记录？

3. 格式化
   └─→ 选择合适章节
   └─→ 使用标准格式
   └─→ 添加元数据 (日期/来源)

4. 写入
   └─→ 读取当前内容
   └─→ 插入新内容
   └─→ 保持结构一致

5. 验证
   └─→ 检查格式正确
   └─→ 检查无重复
   └─→ 检查逻辑一致
```

## 写入模板

### 1. 用户偏好记录
```markdown
### [偏好名称]
**发现日期**: YYYY-MM-DD
**来源**: [对话/观察]

**偏好描述**:
[具体描述]

**应用场景**:
- 场景 1: [如何应用]
- 场景 2: [如何应用]

**优先级**: [高/中/低]
```

### 2. 学习进度记录
```markdown
### [学习主题] 进度
**开始日期**: YYYY-MM-DD
**最后更新**: YYYY-MM-DD

**当前阶段**: [入门/进阶/精通]

**已完成**:
- [ ] 子主题 1
- [x] 子主题 2
- [x] 子主题 3

**下一步**:
- [ ] 待学习主题 1
- [ ] 待学习主题 2

**难点记录**:
- 难点：[描述]
- 解决方案：[如何解决]

**复习计划**:
- 下次复习：YYYY-MM-DD
- 复习方式：[苏格拉底/测试/实践]
```

### 3. 决策记录
```markdown
### 决策：[决策主题]
**决策日期**: YYYY-MM-DD
**决策者**: [用户/共同/助手]

**背景**:
[为什么需要决策]

**选项对比**:
| 选项 | 优点 | 缺点 | 评分 |
|------|------|------|------|
| A | ... | ... | 7/10 |
| B | ... | ... | 9/10 ← |

**最终选择**: [选项 B]
**选择原因**:
1. 原因 1
2. 原因 2

**影响**:
- 短期：[近期影响]
- 长期：[远期影响]

**待执行**:
- [ ] 执行步骤 1
- [ ] 执行步骤 2
```

### 4. 系统配置记录
```markdown
### [系统/工具] 配置
**配置日期**: YYYY-MM-DD
**最后更新**: YYYY-MM-DD

**当前配置**:
```yaml
[配置内容]
```

**变更历史**:
| 日期 | 变更 | 原因 |
|------|------|------|
| YYYY-MM-DD | 初始配置 | 新系统上线 |
| YYYY-MM-DD | 调整 X | 性能优化 |

**注意事项**:
- 注意点 1
- 注意点 2
```

## 更新策略

### 1. 追加式更新
**适用**: 新增信息

```markdown
## 待办事项

### 系统层面
- [x] 已完成事项 (2026-02-01)
- [ ] 进行中事项 (2026-02-05) ← 新增
- [ ] 待开始事项
```

### 2. 替换式更新
**适用**: 信息变更

```markdown
### 当前配置 (2026-02-24 更新)
[新配置内容]

~~旧配置内容~~ ← 可保留历史时注释掉
```

### 3. 归档式更新
**适用**: 信息过期但需保留历史

```markdown
### 历史决策 (已归档)

#### [旧决策] (2026-01-15, 已过期)
[原决策内容]

**归档原因**: 情况变化，被新决策替代
**替代决策**: [链接到新决策]
```

## 记忆维护

### 定期清理 (建议每月一次)

**检查清单**:
- [ ] 移除已过期待办
- [ ] 更新过期配置
- [ ] 合并重复记录
- [ ] 归档历史信息
- [ ] 检查链接有效性

### 记忆压缩

当 MEMORY.md 过大时：

1. **移动细节**: 将详细记录移至 `memory/YYYY-MM-DD.md`
2. **保留摘要**: 在 MEMORY.md 保留核心摘要
3. **添加链接**: 指向详细记录

```markdown
## 学习记录

### AI 开发 (摘要)
**开始**: 2026-02-01
**进度**: 30%
**详情**: `memory/learning-ai-dev.md`
```

## 写入检查清单

写入前确认:
- [ ] 信息真实准确
- [ ] 获得用户同意 (如敏感)
- [ ] 格式符合规范
- [ ] 无重复记录
- [ ] 添加日期和来源

写入后确认:
- [ ] markdown 格式正确
- [ ] 章节结构清晰
- [ ] 链接/引用正确
- [ ] 无语法错误

## 示例对话

**用户**: 记住，我更喜欢用苏格拉底式提问来复习

**助手**:

好的，我会记录这个偏好。

✅ **已记录到 MEMORY.md**

```markdown
### 复习方式偏好
**发现日期**: 2026-02-02
**来源**: 用户明确说明

**偏好描述**:
更喜欢用苏格拉底式提问进行复习，而非单向讲解。

**应用场景**:
- 学习新概念后：用提问检验理解
- 定期复习：用问题引导回忆
- 深度讨论：用问题探索不同视角

**优先级**: 高
```

这样以后每次复习时，我都会用提问的方式帮你巩固知识，而不是直接给答案。
