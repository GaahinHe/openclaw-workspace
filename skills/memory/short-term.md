# 短期记忆管理技能 (Short-Term Memory Management)

## 描述
管理 session 内的上下文，保持对话连贯性

## 使用场景
- 多轮对话中保持上下文
- 追踪当前任务进度
- 记住临时信息和假设
- 管理对话焦点

## 短期记忆结构

### 1. 对话上下文
```
当前 Session:
├── 对话目标 (当前在做什么)
├── 已完成的步骤
├── 待完成的步骤
├── 关键信息 (临时变量、假设)
└── 用户反馈 (修正、偏好)
```

### 2. 任务状态
```
任务追踪:
├── 任务名称
├── 开始时间
├── 当前状态 (进行中/阻塞/完成)
├── 已完成步骤
├── 下一步骤
└── 阻塞原因 (如有)
```

### 3. 临时信息
```
临时记忆:
├── 用户提供的具体值
├── 当前假设
├── 待验证事项
└── 中间结果
```

## 管理策略

### 1. 主动总结
**时机**: 
- 完成一个阶段后
- 对话可能中断时
- 用户长时间未回复

**方式**:
```markdown
## 📝 当前进度

**目标**: [我们正在做什么]

**已完成**:
- [x] 步骤 1
- [x] 步骤 2

**下一步**:
- [ ] 步骤 3

**关键信息**:
- 变量 A = 值 X
- 假设：Y 成立
```

### 2. 关键信息复述
**时机**: 用户使用重要信息时

**方式**:
```
明白了，让我确认一下：
- 你的目标是 X
- 约束条件是 Y
- 优先级是 Z

对吗？
```

### 3. 上下文刷新
**时机**: 
- 对话偏离主题时
- 用户问"我们在做什么"
- 长时间对话后

**方式**:
```
让我们回顾一下：

**原始目标**: [最初的任务]
**当前进展**: [现在在哪里]
**剩余工作**: [还需要做什么]

要继续吗？还是有其他问题？
```

## 上下文保持技巧

### 1. 显式追踪
```python
# 在回复中显式列出状态
当前状态:
- 已分析：用户需求
- 已完成：方案设计
- 进行中：代码实现
- 待完成：测试验证
```

### 2. 隐式引用
```python
# 自然引用之前的内容
如我们刚才讨论的，X 方案的性能更好...
根据你之前提到的需求 Y...
还记得我们发现的 Z 问题吗？
```

### 3. 链接历史
```python
# 引用具体消息
[引用消息 ID 或时间戳]
如你在 10:30 提到的...
根据我们之前的讨论...
```

## 对话焦点管理

### 1. 单焦点原则
**规则**: 一次只处理一个主要话题

**实现**:
```
我注意到我们同时在讨论：
1. X 问题
2. Y 问题

建议先完成 X，再处理 Y，可以吗？
```

### 2. 焦点切换
**当用户切换话题时**:

```
好的，我们先暂停 X 话题，切换到 Y。

**X 话题当前状态**:
- 已完成：[...]
- 待继续：[...]

**Y 话题**:
你想从哪开始？
```

### 3. 焦点恢复
**当用户回到之前话题时**:

```
欢迎回来！继续 X 话题：

**上次进展**:
我们完成了 A 和 B，正准备做 C。

**当前待办**:
- [ ] 完成 C
- [ ] 验证结果

要继续吗？
```

## 临时信息管理

### 1. 分类存储
```
临时信息分类:
├── 配置值 (端口、路径、参数)
├── 用户偏好 (临时性)
├── 假设条件 (待验证)
├── 中间结果 (计算过程)
└── 待确认事项
```

### 2. 过期清理
**识别过期**:
- 任务完成后相关的临时信息
- 被新信息替代的旧信息
- 用户明确说"忘了 X"

**清理方式**:
```
好的，我已经忘记 X 了。

**当前保留的信息**:
- A: [...]
- B: [...]

X 已移除，对吗？
```

### 3. 重要性升级
**当临时信息变重要时**:

```
这个信息很重要，建议记录到长期记忆：

**信息**: [具体内容]
**原因**: [为什么重要]
**用途**: [未来如何使用]

要记到 MEMORY.md 吗？
```

## Session 边界处理

### 1. Session 结束
**当对话自然结束时**:

```
## 📊 本次会话总结

**完成的工作**:
- [x] 任务 A
- [x] 任务 B

**关键决策**:
- 决策 1: [...]
- 决策 2: [...]

**待办事项**:
- [ ] 下次需要做 C
- [ ] 需要确认 D

**建议记录**:
这些信息值得记到长期记忆吗？
- 学习进度 X
- 新发现的偏好 Y
```

### 2. Session 中断
**当对话可能中断时**:

```
## 📌 进度快照

**当前任务**: [任务名称]
**完成度**: 70%

**已完成**:
- [x] 步骤 1-3

**待继续**:
- [ ] 步骤 4: [具体描述]
- [ ] 步骤 5: [具体描述]

**关键上下文**:
- 变量：X = 值
- 假设：Y 成立

下次可以从步骤 4 继续！
```

### 3. Session 恢复
**当新会话开始时**:

```
## 👋 欢迎回来！

**上次会话**: [日期]
**最后进展**: [完成了什么]

**待继续**:
- [ ] 任务 X 的步骤 4
- [ ] 需要确认 Y

要继续之前的工作，还是开始新话题？
```

## 示例对话

**用户**: 帮我写一个排序函数，用 Python，要支持自定义比较

**助手**:

好的，我来帮你实现。

**需求确认**:
- 语言：Python
- 功能：排序函数
- 特性：支持自定义比较函数

我先实现一个基础版本：

```python
def custom_sort(items, compare_fn=None):
    """
    支持自定义比较的排序函数
    
    Args:
        items: 待排序列表
        compare_fn: 比较函数，接受两个参数，返回 -1/0/1
    """
    if compare_fn is None:
        return sorted(items)
    
    # 使用比较函数转换为 key 函数
    from functools import cmp_to_key
    return sorted(items, key=cmp_to_key(compare_fn))

# 使用示例
def compare_desc(a, b):
    """降序比较"""
    if a > b: return -1
    if a < b: return 1
    return 0

result = custom_sort([3, 1, 4, 1, 5], compare_desc)
print(result)  # [5, 4, 3, 1, 1]
```
